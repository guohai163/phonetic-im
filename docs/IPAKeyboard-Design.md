# iOS 15+ 国际音标输入法设计方案

## 1. 目标

- 在 iOS 15+ 提供可安装、可启用的 IPA 输入法。
- 首版优先离线可用，避免依赖“完全访问”权限。
- 输入效率优先于复杂智能：提供常用 IPA 符号分组与快速插入。

## 2. 约束与平台规则

- 形态：`宿主 App + Keyboard Extension`。
- 键盘扩展不能覆盖所有输入场景（例如密码框会回退系统键盘）。
- 无“完全访问”时避免网络请求和跨应用数据共享。
- 需 Apple 签名后安装到真机。

## 3. 功能范围（MVP）

- 3.1 主键盘
  - 4 行主键位，覆盖高频 IPA 字符。
  - Space / Delete / Return / Next Keyboard。
- 3.2 候选条
  - 当前键位可展开变体候选（如 `e -> e ə ɛ`）。
  - 点击候选后直接上屏。
- 3.3 编辑行为
  - 单字符插入。
  - 退格删除。
  - 长按删除（后续版本）。

## 4. 架构

- `IPAKeyMap`: 维护键盘分层与符号集合。
- `IPAComposer`: 轻量输入组合逻辑，输出候选。
- `KeyboardView`: 纯 UI 组件，负责渲染按键和候选。
- `KeyboardViewController`: 与 `textDocumentProxy` 交互，执行插入/删除/换行。

## 5. 迭代计划

- V1（当前）: 静态高频 IPA + 候选变体。
- V2: 双层键盘（元音/辅音/超音段）切页。
- V3: 自定义短语、收藏符号、长按弹出面板。
- V4: 可选云词库同步（需完全访问 + 隐私条款）。

## 6. 测试重点

- iOS 15/16/17 设备兼容。
- 多 App 文本框输入稳定性（Notes, Safari, Messages）。
- 前后台切换后 UI 状态恢复。
- 安全输入场景降级行为。
